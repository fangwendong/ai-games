# 密室逃脱设计笔记

## 产品判断

这个方向和“种花游戏”一样，都是一关一关的结构化闯关，但表达改成“现场找线索 -> 推导密码 -> 开门”。它比纯数独更适合做剧情包装，也更容易在每关塞入不同机制。

## 核心循环

1. 进入一个房间。
2. 观察场景，点击物件拿线索。
3. 推导密码或开锁方式。
4. 输入 / 操作 / 解锁。
5. 门开后进入下一关。

## 关卡原则

- 每关只讲一个新机制，避免玩家一关学太多东西。
- 线索要足够直接，不能靠盲猜。
- 每关的开门方式要和场景元素一致，比如钟表、书架、开关、画框、颜色、字母等。
- 失败后的提示要明确，不要只说“错了”。

## 当前原型关卡类型

- 数字密码
- 顺序密码
- 字母转换
- 开关顺序
- 镜像 / 方向提示
- 分组计数
- 组合锁
- 最终混合关

## 下一步测试

- 新手是否能在 10 秒内理解“点物件找线索”
- 每关平均卡住时间
- 提示是否足够，不会让玩家直接放弃
- 是否需要加入道具或限时挑战
